# 02 PRD

## 1. 项目名称

专家邀约后专家端申报流程

## 2. 项目目标

为收到邮件邀约的全球专家提供一个清晰、可恢复、可异步处理的在线申报流程，完成从项目介绍、简历初筛、缺失信息补充、证明材料上传到最终提交的闭环。

## 3. 范围说明

### 3.1 本次范围

- 专家访问链接后的页面流程
- 专家身份识别
- 简历上传与分析结果反馈
- 缺失字段补填与二次分析
- 材料上传与最终提交
- 进度恢复

### 3.2 不在本次范围

- 邮件发送与邀约名单管理
- 后台人工审核工作台
- 邮件/短信通知
- 多语言支持
- 运营数据报表

## 4. 目标用户

- 主要用户：收到邀约邮件的全球专家
- 用户特征：
  - 可能跨设备、多次进入
  - 不熟悉申报流程
  - 对等待无反馈较敏感

## 5. 核心用户旅程

1. 专家通过带 `token` 的链接进入系统
2. 查看 GESF / 全球引才项目介绍
3. 上传简历
4. 系统异步分析简历并反馈状态
5. 系统给出 3 类结果：
   - 不符合申报资格
   - 信息不足
   - 初步符合
6. 若信息不足，专家补充结构化字段并触发二次分析
7. 若符合，专家进入材料上传页，按 6 类材料补充上传
8. 专家确认提交
9. 系统提示已收到材料信息，将在 1-3 个工作日内答复
10. 专家再次打开链接时恢复之前进度

## 6. 页面需求

### 6.1 项目介绍页

- 展示项目背景、价值、流程概览
- 提供 `开始申请` 按钮

### 6.2 简历上传页

- 支持上传简历
- 展示上传要求
- 上传成功后发起分析

### 6.3 分析中页

- 展示分析进度文案
- 支持刷新后恢复状态

### 6.4 分析结果页

- 展示三态结果：
  - `INELIGIBLE`
  - `INFO_REQUIRED`
  - `ELIGIBLE`

### 6.5 信息补充页

- 展示结构化缺失字段
- 支持补填并再次分析

### 6.6 材料上传页

- 包含 6 个材料分类
- 每类支持批量上传
- 允许留空
- 支持回显已上传材料

### 6.7 提交确认

- 点击确认后提交全部材料
- 弹窗提示：`已收到材料信息，将在 1-3 个工作日内答复。`

### 6.8 异常页

- token 无效
- token 过期
- 服务异常
- 上传失败

## 7. 功能需求

### 7.1 身份识别与会话初始化

- 通过邮件唯一 `token` 识别专家
- 初始化或恢复申请记录
- token 无效时禁止进入流程

### 7.2 简历分析

- 上传简历后创建分析任务
- 调用既有简历分析服务
- 前端查询分析状态

### 7.3 资格结果判断

- 判断逻辑来源：大模型 + 固定提示词规则
- 结果类型：
  - 不符合
  - 信息不足
  - 符合

### 7.4 缺失字段补填

- 后端返回结构化 `missing_fields`
- 前端动态渲染字段表单
- 补填后触发二次分析

### 7.5 材料上传

- 分类：
  - 身份证明
  - 工作证明
  - 学历证明
  - 荣誉证明
  - 专利证明
  - 项目证明
- 每类支持批量上传
- 可删除、补传、回显

### 7.6 进度恢复

- 服务端保存当前状态
- 专家再次访问时自动恢复

## 8. 用户故事与验收标准

### 8.1 身份识别

As a 专家, I want the system to recognize me from the invite link, so that I can continue without registering again.

验收标准：

- token 有效时可识别对应专家
- 若已有申请，则自动恢复当前步骤
- token 无效时展示异常提示页

### 8.2 简历上传与反馈

As a 专家, I want clear feedback after I upload my CV, so that I do not wait on a blank page.

验收标准：

- 上传成功后立即进入分析中状态
- 处理期间页面有状态反馈
- 关闭后再次进入仍能看到当前处理状态

### 8.3 信息不足补填

As a 专家, I want the system to tell me exactly which fields are missing, so that I can provide only what is needed.

验收标准：

- 缺失字段以结构化方式返回
- 支持补填后重新分析
- 二次分析后返回新的结果

### 8.4 材料上传

As a 专家, I want to upload supporting materials by category, so that I can complete the application in multiple sessions.

验收标准：

- 6 类材料可分别上传
- 已上传文件可回显
- 材料页允许分次完成

### 8.5 最终提交

As a 专家, I want confirmation after submission, so that I know my materials were received.

验收标准：

- 点击提交后状态变为 `SUBMITTED`
- 页面弹出提交成功提示
- 已提交申请再次进入时显示已提交状态

## 9. 非功能要求

- 支持桌面端与移动端常见浏览器
- 核心状态与文件记录必须持久化
- 敏感文件需受控访问
- 关键操作需可审计

## 10. 成功标准

- 专家可从链接进入并完成完整提交流程
- 中断后可恢复到正确步骤
- 三态结果能驱动后续页面流转
- 缺失字段补填后二次分析可闭环
