# 06 Implementation Plan

## 1. 实施策略

采用适合 Vibe Coding 的纵向切片方式实施，每一轮都尽量做到前后端联通、状态可验证、文档可回写。

## 2. 切片规划

### Slice 1: token 识别与申请初始化

目标：

- 支持通过 `token` 进入系统
- 创建或恢复申请记录
- 完成介绍页与申请快照接口

完成标准：

- 访问链接即可识别专家
- token 无效时返回异常页
- 可返回当前申请状态

### Slice 2: 简历上传与分析中状态

目标：

- 支持简历上传
- 创建分析任务
- 分析中页可轮询状态

完成标准：

- 上传成功后状态转入 `CV_ANALYZING`
- 分析状态接口可正常工作

### Slice 3: 分析结果三态

目标：

- 接入既有简历分析结果
- 完成三态结果页

完成标准：

- 能正确进入 `INELIGIBLE`
- 能正确进入 `INFO_REQUIRED`
- 能正确进入 `ELIGIBLE`

### Slice 4: 结构化补填与二次分析

目标：

- 渲染 `missing_fields`
- 提交补充字段
- 触发二次分析

完成标准：

- 可动态渲染字段
- 二次分析后返回新结果

### Slice 5: 材料上传

目标：

- 实现 6 类材料上传与回显
- 支持分批保存

完成标准：

- 每类可批量上传
- 已传材料可回显与删除

### Slice 6: 最终提交与已提交恢复

目标：

- 实现最终提交
- 显示提交成功提示
- 再次访问时显示已提交状态

完成标准：

- 提交接口幂等
- 状态转为 `SUBMITTED`

## 3. 每个切片的标准执行步骤

1. 先重读相关 PRD 段落
2. 确认状态流转影响
3. 只读取与当前切片相关的文件
4. 先改接口和数据，再改页面
5. 先跑最小验证，再继续下一个切片
6. 回写文档变化

## 4. 联调顺序建议

1. `expert-session`
2. `resume upload`
3. `analysis-status`
4. `analysis-result`
5. `supplemental-fields`
6. `materials`
7. `submit`

## 5. 交付节奏建议

- 第 1 周：Slice 1-2
- 第 2 周：Slice 3-4
- 第 3 周：Slice 5
- 第 4 周：Slice 6 + 联调 + 提测
