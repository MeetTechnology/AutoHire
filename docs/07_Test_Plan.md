# 07 Test Plan

## 1. 测试目标

验证专家端流程可完整运行，状态恢复准确，分析结果驱动正确，材料提交可靠。

## 2. 核心测试范围

- token 鉴权
- 申请初始化与恢复
- 简历上传
- 分析中状态反馈
- 首次分析三态结果
- 缺失字段补填与二次分析
- 材料上传、删除、回显
- 最终提交
- 异常处理

## 3. 关键用例

### 3.1 正常主路径

- 有效 token 进入介绍页
- 上传简历
- 返回符合资格
- 上传部分或全部材料
- 点击提交
- 页面提示提交成功

### 3.2 信息不足路径

- 上传简历
- 返回 `INFO_REQUIRED`
- 显示结构化字段
- 补填后重新分析
- 返回 `ELIGIBLE`
- 上传材料并提交

### 3.3 不符合路径

- 上传简历
- 返回 `INELIGIBLE`
- 展示原因
- 流程结束

### 3.4 进度恢复

- 在分析中关闭页面后重新打开
- 在材料上传中关闭页面后重新打开
- 提交后重新打开

### 3.5 异常场景

- token 无效
- token 过期
- 简历上传失败
- 分析任务失败
- 材料上传部分失败
- 重复点击提交

## 4. 验收重点

- 页面跳转与后端状态一致
- 任何关键步骤刷新后都可恢复
- 缺失字段表单类型与后端定义一致
- 已提交状态不可错乱

## 5. 建议测试层次

- 单元测试：
  - 状态流转
  - 字段映射
  - 接口参数校验
- 集成测试：
  - 上传到分析结果链路
  - 补填后二次分析链路
  - 材料上传与提交链路
- 端到端测试：
  - 正常主路径
  - 信息不足路径
  - 进度恢复路径
