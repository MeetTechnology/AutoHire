# 08 Risks & Open Questions

## 1. 已确认决策

- 采用 `固定主链接 + 专家唯一 token`
- 资格判断使用“大模型 + 固定提示词规则”
- 缺失信息采用结构化字段补填
- 1-3 个工作日答复仅做页面提示
- 简历处理能力复用既有项目

## 2. 当前主要风险

### 2.1 token 被转发或泄露

风险：

- 他人获取链接后可能访问专家数据

建议：

- token 设置失效机制
- 关键页面增加二次身份确认能力作为预案

### 2.2 既有分析服务返回结构不稳定

风险：

- 前端动态表单无法稳定渲染

建议：

- 在适配层统一输出标准结构

### 2.3 大模型分析耗时过长

风险：

- 专家等待体验差

建议：

- 强制进入分析中页
- 明确展示处理中状态
- 支持失败重试

### 2.4 材料上传量大

风险：

- 单次上传失败率升高

建议：

- 分类分批上传
- 单文件失败不影响其他文件

## 3. 仍待业务确认的问题

- token 是否需要有效期限制
- 已提交后是否允许后台触发补件
- 各材料分类是否存在后续的“推荐项/必填项”规则
- 页面是否需要明确展示当前步骤导航

## 4. 上线前检查清单

- token 鉴权已联调通过
- 简历分析适配接口返回结构已冻结
- 文件上传与受控下载已验证
- 状态恢复在多次刷新下表现稳定
- 提交接口幂等已验证
